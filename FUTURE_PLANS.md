# Future Plans

## 1) Data Expansion (Highest Priority)

- Complete Grade 10–12 subject/year/month paper coverage
- Standardize naming and metadata for all imported papers/memos
- Add manifest validation checks before assets are used in-app

## 2) Question-Mapping Accuracy

- Improve support for:
  - multi-page single questions
  - multiple questions on one page
  - memo sections that do not align linearly with question pages
- Introduce optional crop metadata workflow for mixed pages

## 3) Guided Help

- Replace `Ask Tutor` placeholder with provider-agnostic integration layer
- Add prompt/context guardrails to keep answers aligned to memo scope
- Add offline fallback messaging when help services are unavailable

## 4) Library and Content Layer

- Add curated notes per subject/topic
- Add embedded video resources linked by curriculum topics
- Add quick-revision bundles for high-frequency exam areas

## 5) Offline and Performance

- Expand download/cache controls for papers and memo assets
- Improve memory usage for long paper sessions
- Add periodic cleanup policy for stale cached resources

## 6) Quality and Delivery

- Add automated checks for:
  - manifest integrity
  - script pipeline success/failure reporting
  - Android build health in CI
- Establish repeatable release process with versioning and changelog policy

## 7) Team and Community Readiness

- Add issue/PR templates
- Publish contribution standards for data and UI changes
- Define documentation ownership for operational guides
