# Security Policy

## Supported Scope

Security reports are accepted for:

- Android app code under `app/`
- Python automation scripts under `scripts/`
- Data-processing logic affecting integrity of generated manifests/assets

## Reporting a Vulnerability

Please report vulnerabilities privately by opening a private security report on your Git hosting platform (or direct private contact with the maintainer) instead of posting public issues.

Include:

1. A clear description of the vulnerability
2. Reproduction steps or proof-of-concept
3. Impact assessment (what can be compromised)
4. Suggested mitigations (optional)

## Response Expectations

- Initial acknowledgement target: within 7 days
- Triage and severity assessment: as soon as reproducible
- Fix timeline: depends on severity and complexity

## Disclosure

Please allow reasonable time for remediation before public disclosure.
